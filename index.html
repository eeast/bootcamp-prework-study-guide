HTML

The Basics

  • HTML is used to create the structure of a webpage.
  • Webpages are made up of elements which work together to provide content and the basic framework for all pages on the web.
  • An element is a type of content on a webpage.
  • Common element types used to add content to a page include "Text", "Image", "Header" and "List" elements.
  • Common element types used to divide and describe a page include "Meta-related", "Content Division", and "Semantic" elements
  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • Section elements are semantic elements that define a section in a document that contains thematic content.
  • "Elements" refers to the entire element from opening tag to closing tag.
  • "Tag" refers only to what appears inside the angle brackets at the start and end of an element.

Common Elements

  • <html> - represents the root, or start, of our HTML document. Generally includes the lang metadata element.
  • <head> - contains information (metadata other than lang) that helps the browser to render the page correctly. Elements included within this element are not visible when a user looks at a webpage.
  • <title> - defines the title for the webpage. This element is required and is extremely important for SEO.
  • <style> - defines the CSS styles associated with a website.
  • <base> - defines the base URL for a webpage.
  • <link> - connects an external resource to the HTML document.
  • <meta> - defines metadata such as the character set, description, keywords, author, and viewport.
  • <script> - embeds client-side scripts in the HTML document.
  • <body> - contains the content that is shown to the user. All the elements that we want the user to see and interact with must be inside this element.
  • <div> - used to create a division in the page.
  • <span> - used to create an inline division.
  • <br> - used to create a line of empty space, or a line break, between two blocks of content.
  • <p> - represents a paragraph of block of text.
  • <ul> - represents an unordered list; each item is led by a bullet point.
  • <ol> - represents an ordered list; each element is led by a number, in sequential order.
  • <li> - represents an item in either an ordered or unordered list.
  • <img> - contains information about images that are displayed on the webpage. Attributes typically include at least: src, alt, width, and height.

Metadata Elements

  • lang - tells the browser the language for the webpage ("en" for English)
  • charset - sets our character encoding (UTF-8 is most common).
  • http-equiv - sets an HTTP header for content.
  • viewport - sets the default view for our webpage.
  • src - defines the location of the image to be shown.
  • alt - provides alternative text to be shown when an image can not be loaded.
  • width - sets the image width in pixels on the webpage.
  • height - sets the image height in pixels on the webpage.

CSS

The Basics

  • CSS (Cascading Style Sheet) is a common way to add color, structure, and other formatting to a webpage.
  • There are three ways to style a webpage using CSS: inline CSS, internal CSS style sheet, and external CSS style sheet.
  • CSS contains two main parts: the Selector, and the Declarations. The Declaration contains two parts: the Property, and the Value. (sel {property: value;})
  • External CSS style sheets are the preferred way for adding CSS to a webpage. This allows for "Separation of Concerns", or limiting each file to one aspect of the webpage.

Common CSS Properties

  • color: - changes the color of the selected item. Accepts words of common colors, such as "blue".
  • margin: - indicates how much space we want around the outside of an element.
  • padding: - indicates how much space we want around the content inside the element.


Git

The Basics

  • Git is frequently used in software development to facilitate sharing of code and implementation of version control.
  • This process is preserved through the Git Workflow, which uses branches as working areas for making simultaneous changes to code.
  • Each branch should be properly labeled with a detailed title identifying what the object for a particular branch is.
  • When changes have been completed, this branch can be pushed safely to the Git repository, where it can be flagged for review and safely merged into the main (sometimes called "master" branch).
  • It is important to pull often from the Git source to ensure any changes made to the main branch are reflected in your working branch.

Common Git Commands

  • git status - returns the current working branch and any changes reflected on the local machine.
  • git checkout -b [branch-name] - moves the current working branch to a new branch.
  • git add -A - adds any modification in the current working branch to the staging area.
  • git commit -m "message" - commits changes in the staging area to the repository.
  • git pull origin [branch-name] - pulls changes from the branch on the repository to the local machine.
  • git push origin [branch-name] - pushes changes from the local machine to the branch on the repository.

Common Git Flags

  • -b - indicates a branch; followed by the branch name, i.e. "feature/add-html".
  • -A - indicates "All", i.e. git add -A would add all changes to the staging area.
  • -m - indicates a message; followed by the message content, i.e. "added html"

JavaScript

The Basics

  • JavaScript is a commonly used scripting language for adding functionality to a webpage.
  • Through the use of JS, you can add impressively complex computations and functionality that makes a webpage significantly more engaging.
  • The starting building blocks of JavaScript are variables, loops, and functions.
  • There are many different data types that a variable can be. Also, JS does not require a variable to have a declared type; rather, the data type is assigned based on the value saved.
  • Loops allow us to repeat lines of code in succesion. This is particularly useful when using an incrementing variable in a data structure such as an array.
  • Functions can be defined in our code, and allow us to run a specific subset of code at any time by using the Function Call.

Common Variable Types

  • String - A string of text; must be surrounded by double quotation marks (" ").
  • Number - Any integer less than the max integer limit (9007199254740991).
  • Bigint - An integer larger than the max integer limit.
  • Boolean - A simple data type that contains either true or false.
  • Undefined - Default data type for variables that have not been initialized or are missing another default value.
  • Null - Represents the absense of data; Not zero, but nothing. Zero is something.
  • Symbol - A specific data type that's guaranteed to be unique.
  • Object - A standalone entity, with properties and type.

Function Basics

  • There are many terms used with functions that are common between languages that use functions, such as:
    • "Name" - the name given to the function. This will be used to call the function later.
    • "Call" - term used to describe the invocation of a function. This is done by typing the function name and any required arguments after the function has been defined.
    • "Definition" - the original declaration and creation of the function. Must be placed first in the code before any function calls.
    • "Argument(s)" - Variables that are included in the parentheses of a function call. Are specified in the function definition.
  • Functions must be declared as such with the function keyword.
  • Functions must include parentheses after the function name (i.e. function myFunction() {} ).
  • Arguments can be specified in the function definition, and will be required to be passed in for every function call.
